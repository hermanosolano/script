Após logar na nossa conta, vamos para OVERVIEW que é a visão geral de nosso repositório, quando tivermos usado e abusado do Github vamos visualizar os nossos repositórios populares, de acordo com o uso que tivermos deles; as contribuições feitas durante o ano mostrado num gráfico, as atividades de contribuição, mostra também quando você faz o seu primeiro PULL REQUEST (que é um pedido de solicitação), a primeira vez que editamos algum arquivo do nosso repositório, assim como mostra qual foi o seu primeiro repositório e quando você se juntou ao Github.

No canto esquerdo superior podemos ver o nosso perfil, podemos personalizá-lo inserindo uma foto, uma descrição sobre nós e inserir um status que dirá sobre a nossa disponibilidade atual no Github.

No canto direito temos CUSTOMIZE PINNS onde temos a possibilidade de fixar itens para ficar em destaque no perfil, por exemplo você fixa seus repositórios mais relevantes para mostrar para outras pessoas que visitarem seu perfil. É uma ótima forma de mostrar os seus projetos mais relevantes.

Em REPOSITORIES é a listagem de nossos repositórios. Repositório é o elemento mais básico do Github e é nele que vai ficar o nosso projeto que pode conter pastas, imagens, arquivos, incluindo a documentação e armazena o histórico de revisões de cada arquivo. Podemos ter vários colaboradores e os repositórios podem ser públicos ou privados. Colaborador é o usuário que contribui em outro repositório no qual não foi criado por ele. Mantenedor é o usuário que criou o repositório e quando precisa de contribuições solicita colaboradores.

Ao lado direito encontramos um botão NEWpara criação de um novo repositório que também é representado pela URL https://github.com/new. Ou também podemos criar observando o lado superior direito num ícone + e, em seguida, clicamos em NEW REPOSITORY. De qualquer modo, inserimos um nome ao nosso novo repositório, uma breve descrição sobre o nosso projeto, clicamos na caixa de seleção INITIALIZE THIS REPOSITORY WITH A README que é um arquivo que deve conter informações sobre o nosso projeto. Observe que a descrição que colocamos acima incluirá no README, mas nada afetaria caso não clicássemos na caixa de seleção.

Enfim, nosso novo primeiro repositório se encontrará em https://github.com/username/novo_repositorio. Observe que temos mais abas para explorar, mas antes vamos explorar essa página inicial de nosso repositório que contém alguns elementos:

  • Watch: se você quer acompanhar um repositório específico, isto é, “assistí-lo" para saber todas as suas atualizações, podendo personalizar de algumas formas para isso;

  • Stars: dando uma estrela você opta por acompanhar o desenvolvimento do repositório em questão. É uma forma de mostrar também que você “curtiu” um repositório e salvá-lo na sua preferência e, assim, achar assuntos relacionados no seu feed. Facilitando a localização do repositório também. Você pode ver todos os repositórios e tópicos estrelados ao acessar https://github.com/stars. Quanto mais estrelas (stars) recebe um repositório mais ele fica ranqueado no Github e com isso é mais visto, a depender do tópico relacionado também. Além disso, o Explore mostra repositórios populares com base no número de estrelas que eles têm.

  • Fork: significa dar uma "garfada" no repositório de alguém para trabalharmos nele; é a mesma coisa que você fazer uma cópia e enviar para o seu Github. Sendo assim, qualquer alteração feita no seu repositório não irá alterar nada no repositório original. Bifurcar um repositório é semelhante a copiar outro repositório, com duas diferenças principais:

    • (i) você pode usar um pull request para sugerir alterações de seu fork para o repositório original, também conhecido como repositório upstream, e;

    • (ii) você pode trazer alterações do repositório upstream para o fork local, sincronizando seu fork com o repositório upstream.

Debaixo da nossa descrição, feita anteriormente na criação do nosso repositório, encontra-se MANAGE TOPICS, onde será nele que inseriremos os tópicos relacionados ao nosso repositório.

Tópicos: uma forma de categorizar seus repositórios. Através deles podemos explorar outros repositórios de mesmo tópico, assim encontrando projetos relacionados só clicando nos tópicos em questão depois de tê-los criado ou para navegar pelos tópicos mais usados, acesse: https://github.com/topics/.

Mas abaixo temos termos horizontalmente organizados: commits, branch, releases e contributor. São informações relevantes referente ao nosso repositório.

  • COMMIT: significa literalmente “confirmar”. Qualquer ação feita no Github é baseado em fazer para depois confirmar, então basicamente commit significa isso, confirmar as alterações feitas no repositório, em algum arquivo específico. Cada commit tem uma mensagem associada para descrevermos o que foi feito.

  • BRANCH: significa ramo, ramificação ou podemos chamar de filial. Como desenvolvedores já sentimos a necessidade em trabalhar em versões diferentes de um arquivo, então o objetivo do branch é exatamente esse de trabalhar em diferenfes versões de um repositório de uma só vez. A ramificação pode ser considerada uma versão paralela de um repositório. Existe uma ramificação padrão/principal chamada de master (branch master) ou ramificação definitiva, mas podemos criar ramificações secundárias. Qualquer ramificação criada será uma cópia da master. Criar outras ramificações (ou branches) permite que você trabalhe livremente sem sobrepopor o que você já tinha feito em outra ramificação. Para salvar as alterações feitas no novo ramo para o ramo master fazemos o processo que chamamos de mesclagem (merge) através do pull request. Use uma ramificação para isolar o trabalho de desenvolvimento sem afetar outras ramificações no repositório. Você pode usar ramificações para:

    1. Desenvolver recursos;
    2. Corrigir erros;
    3. Experimentar com segurança novas ideias.

    Resumindo, qualquer coisa que você fizer no seu repositório, crie uma ramificação, faça o que for necessário só para depois mesclar (unir) para a ramificação principal (master) através do pull request, isso tanto no seu próprio repositório assim como nos alheios. PULL REQUEST poderia ser traduzido como "pedido de solicitação", então crie um pull request para propor e colaborar em alterações em um repositório. Essas alterações são propostas em uma ramificação, como já vimos, o que garante que a branch master contenha apenas trabalhos concluídos e aprovados.

  • CONTRIBUTOR: mostra quem são os contribuidores do repositório em questão.
